## 연습문제

1. 요구 수집 단계의 주요 업무가 아닌 것은? -> 4
  1. 수용범위결정
  2. 기존문서조사
  3. 사용자요구사항수집
  4. 범기관적 제약조건 파악

2. 요구 수집 명세서를 작성할 때 포함해야 할 항목에 해당하지 않는 것은? -> 2
  1. 수집한세부내용을포함하는첨부자료
  2. 업무영역분할도
  3. 정보제공자
  4. 수집담당자

3. 요구 분석 단계의 주요 업무가 아닌 것은? -> 3
  1. 업무영역분할
  2. 상세업무분석
  3. 트랜잭션 요구 분석 명세서 작성
  4. 사용자그룹및담당자식별

4. 요구 분석이 어려운 이유가 무엇인지 설명하시오. 
-> 개발자와 사용자의 원활한 의사소통이 이루어지기 어렵고, 요구사항이 계속 변화하기 따름이고, 문서화 또한 어렵다.

5. 요구 수집을 위한 인터뷰 시 유의 사항으로 옳지 않은 것은? -> 4
  1. 인터뷰 대상의 기분을 상하게 하는 일이 없도록 언행에 유의한다.
  2. 인터뷰는 반드시 정해진 시간 안에 마치도록 한다
  3. 닫힌 질문과 열린 질문을 적절히 혼합해서 활용한다.
  4. 인터뷰할 내용을 인터뷰 대상자에게 미리 알리면 선입견을 갖게 되므로 유의한다.

6. DB 구축의 전체 단계에 걸쳐서 요구 분석 단계가 가장 중요한 이유가 무엇인지 설명하시오.
-> 요구 분석 단계에서 잘못되면 이후에 수정이 매우 어렵기 때문이다. 

7. 다음 중 요구사항의 특정에 해당하지 않는 것은? -> 2
  1. 요구사항은 계속 변화한다.
  2. 요구사항은 완전하다.
  3. 요구사항은 라이프사이클이 있다.
  4. 요구사항은 점차확장된다.

8. 요구사항분석 시 고려 사항으로 옳지 않은 것은? -> 4
  1. 현시점에 펼요한 요구사항에 초점을 맞추어 분석해야한다.
  2. 요구사항의 일관성이 보장되도록상반되는요구사항은조정해야한다.
  3. 요구사항이 상충되는 경우 상호 협의를 통해서 요구사항을 단일화해야한다.
  4. 업무담당자와주요사용자도요구분석 명세서를함께 검토해야한다.

9. 다음 중 데이터 요구 분석 명세서를 작성할 때의 유의 사항으로 옳은 것은? -> 2
  1. 해당 분야의 전문적인 용어를 사용해서 작성한다.
  2. 가능하면 구체적인 표현을 시용한다.
  3. 단순한 표준 문체를 사용한다.
  4. 기능하면 용어를 서로 참조하지 않도록 작성한다.

10. ( ) 안에 적합한용어는?
  - 수집된 사용자의 요구사항을 분석해서 시용자와 DB 개발자를 연결하는 중요한 문서가 되는(________)를 정확히 작성하는 것이 중요하다.
 -> 요구 분석 명세서

11. 다음 중 트랜잭션 요구 분석 명세서를 작성할 때의 유의 사항으로 잘못된 것은? -> 4
  1. 한 번에 하나 이상의 데이터 조작이 포함하도록 작성한다.
  2. 데이터 조작 작업은 되도록 검색， 삽입 ， 삭제， 변경이라는 용어를 시용해서 표현한다.
  3. 데이터조작작업의주체가누구인지를명확하게작성한다.
  4. 분할된 업무 영역별로요구되는데이터 조작작업을간단히 표현한다.
